# Learning Tool 📚

Личная оффлайн-платформа для учёбы: подготовка к ЕГЭ (информатика, профильная математика, русский), Golang, английский и Computer Science.

Без сборки, без сервера, без интернета. Открыл файл — учишься.

## Как запустить

**Самый простой способ** — открыть `index.html` двойным кликом. Всё работает.

**Способ получше** (включается оффлайн-кэш, установка как приложение, и практика с кодом — см. ниже):

```
python -m http.server 8000
# открыть http://localhost:8000
```

> ⚠️ Раздел «Практика» (редактор кода с проверкой тестов) работает **только** через `http.server` — Pyodide грузит свой рантайм через `fetch()`, а браузеры блокируют `fetch()` для файлов, открытых напрямую (`file://`). Всё остальное на платформе прекрасно работает и с `file://`.

## Доступ с телефона

1. Запусти на ПК `python -m http.server 8000`.
2. Узнай IP компьютера в домашней сети: `ipconfig` (Windows) → строка IPv4, например `192.168.1.42`.
3. На телефоне (в том же Wi-Fi) открой `http://192.168.1.42:8000` и добавь на главный экран.

Важно: прогресс на телефоне и на ПК хранится раздельно (localStorage у каждого устройства свой). Перенести можно экспортом/импортом JSON из «Настроек». Вариант проще — включить GitHub Pages в репозитории, тогда сайт доступен отовсюду по одной ссылке.

## Как устроено

```
index.html            — страница приложения
css/style.css         — оформление
js/app.js             — вся логика (роутер, прогресс, статистика, редакторы)
content/manifest.js   — схема курса + регистрация контента
content/*.js          — материал курсов (по файлу на направление)
files/                — файлы-приложения к урокам (напр. официальные файлы ФИПИ для файловых заданий ЕГЭ)
js/vendor/             — сторонние библиотеки, вендоренные локально (без CDN, работают офлайн):
  prism/                  — подсветка синтаксиса в блоках кода
  codemirror/             — редактор кода на странице «Практика»
  pyodide/                — Python (CPython → WebAssembly), выполняет код студента прямо в браузере
  jsxgraph/               — интерактивные графики/геометрия/чертежи в уроках (аналог GeoGebra)
sw.js                 — сервис-воркер для оффлайна (нужен только по http)
```

- **Материал** лежит в `content/*.js`. Claude добавляет уроки коммитами → `git pull` → обновить страницу.
- **Твои правки и свои курсы** делаются прямо в браузере (кнопки «＋» и «Редактировать») и хранятся в localStorage.
- **Прогресс и статистика** (посещения, стрик, % прохождения, статусы ДЗ) — тоже в localStorage.
- **Экспорт/импорт** всего в JSON — в «Настройках». Делай бэкап время от времени!
- **Карточки для повторения** — у уроков есть флеш-карточки; кнопка «🃏» копирует их в формате RemNote (`вопрос >> ответ`, при вставке сами становятся карточками) или Anki (TSV).
- **«Планирую узнать»** — вкладка-копилка: кидай туда видео, статьи и темы на будущее; у каждой есть кнопка «🤖», которая собирает запрос Claude «сделай из этого урок». Ютуб-ссылки показываются лентой с превью и длительностью, название подтягивается автоматически (когда есть интернет).
- **Автосохранение на диск** — в настройках можно выбрать папку, и копия всех данных будет писаться в JSON-файл при каждом изменении (Chrome/Edge). Плюс всё всегда сохраняется в localStorage автоматически.
- **Практика с кодом** — у части уроков есть кнопка «🧑‍💻 Практика»: условие задачи + редактор Python (подсветка синтаксиса, как в Obsidian) + кнопка «Запустить на всех тестах», всё на одном экране, без переключения вкладок браузера. Код выполняется прямо в браузере через Pyodide (Python, скомпилированный в WebAssembly) — интернет не нужен, сторонние серверы не участвуют. Есть кнопка «Показать эталонное решение». Код студента автосохраняется по мере набора.
- **Интерактивные графики/чертежи** — в markdown урока можно вставить живой график функции, чертёж с перетаскиваемыми точками (геометрия), диаграмму графа и т.д. — блоком ` ```jsxgraph `. Работает как GeoGebra, но локально, без интернета и без своего приложения.
- **🔁 Повторение** — отдельная вкладка в шапке. Каждое успешное прохождение практики планирует следующее повторение (1 → 3 → 7 → 14 → 30 дней, чем увереннее решаешь — тем реже повторяем). На вкладке — что просрочено сегодня, что скоро, и ссылка на официальный каталог заданий (Решу ЕГЭ) за свежими вариантами того же типа.

## Рабочий цикл с Claude

1. Учишь урок, делаешь ДЗ.
2. Кнопка **«📤 Сдать ДЗ Claude»** — собирает задание + твоё решение в текст, копируешь и отправляешь в чат.
3. Кнопка **«🤖 Спросить Claude»** — быстрый вопрос с контекстом урока.
4. Нужен новый материал — просишь в чате, Claude коммитит в `content/`, ты делаешь `git pull`.

Токены тратятся только на проверку ДЗ и вопросы — сама платформа работает без Claude.

## Как добавить урок руками

Формат описан в `content/manifest.js`. Коротко: в файле курса в нужный модуль добавляется объект

```js
{
  id: "go-06",                 // уникальный id внутри курса
  title: "Название урока",
  theory: `материал в markdown`,
  homework: `текст ДЗ`,        // необязательно
  cards: [                     // необязательно — карточки для RemNote/Anki
    { q: "вопрос", a: "ответ" }
  ],
  practice: [                  // необязательно — задачи в редакторе кода (см. content/manifest.js)
    {
      id: "p1", title: "Название вкладки",
      statement: `условие в markdown`,
      starterCode: `# заготовка\n`,
      tests: [{ input: "5\n3\n", expected: "8" }],
      solution: `эталонный python-код`
    }
  ]
}
```
